Master Thesis at Autodesk

I am doing my Master Thesis project together with Martin Engelbrektsson at Autodesk in Stockholm. We are developing a plug-in for telemetry visualization for the game engine Stingray. Our hopes are that such a plug-in can help game developers and designers find performance and design bottlenecks. We are cooperating with Fatshark and have recieved several gigabytes of telemetry data recorded from real sessions in the game Vermintide.

The plugin consists of an editor and and engine part that communicates back and forth via Lua. The editor front-end is written in JavaScript with the Mithril.js API, which is designed for building HTML based single-page applications. A custom DLL written in C performs heavy database related tasks for the editor. The MongoDB C driver is used for database communication. The editor front-end uses Lua and a C backend for more heavy tasks. More heavy tasks such as extracting data from the database utilizes a custom DLL:s written in C.

As the project is still in progress, more info and images of our results will come soon!

C C++ Javascript Lua MongoDB Mithril Visualization Front-end Back-end

Multiplayer System for Mobile Units

As Batchelor's Thesis me and six others created an entertainment system with the purpose of making players get together and play video games on a tablet with their smartphones as controllers. The project was developed with Java using the framework LibGDX for graphics and gameplay and KryoNet for networking. The system can be run on any iOS, Android or desktop application. It comes with one pre-installed game Krazy-Razy, which we also developed.

Scrum was used in order to get the whole team working efficiency. I mostly worked with the general structure of the system, the controller system and UX.
